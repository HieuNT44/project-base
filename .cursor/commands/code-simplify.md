# Code simplify — giảm phức tạp

Invoke **`code-simplification`** skill.

1. Preserve exact behavior — run tests before and after
2. Chesterton's Fence — understand why code exists before removing
3. Rule of 500 — prefer readable units
4. Output: simplified diff only; no scope creep

Pair with **`code-review-and-quality`** if the change is large.
